# 游戏开发后续任务

## 任务目标
让AI总结游戏开发上下文，并提取值得作为伏笔的细节，为后续游戏剧情发展做准备。

## 任务步骤

### 1. 上下文总结
- [ ] 回顾游戏开发的完整历程
- [ ] 总结已实现的功能和技术细节
- [ ] 梳理游戏的核心机制和设计理念
- [ ] 记录当前游戏的状态和限制

### 2. 伏笔细节提取
- [ ] 分析上下文，识别潜在的剧情线索
- [ ] 检查聊天记录中的关键对话，寻找可扩展的情节
- [ ] 梳理游戏机制中的隐藏可能性
- [ ] 识别技术实现中的可扩展点

### 3. 伏笔设计与规划
- [ ] 将提取的细节分类为不同类型的伏笔（剧情、机制、角色等）
- [ ] 设计伏笔的后续发展方向
- [ ] 规划伏笔的揭示时机和方式
- [ ] 确保伏笔与游戏核心主题一致

### 4. 文档整理与输出
- [ ] 将总结和伏笔细节整理成结构化文档
- [ ] 为每个伏笔添加详细的发展建议
- [ ] 标注伏笔的优先级和重要性
- [ ] 输出最终的伏笔设计文档

## 预期结果

1. **上下文总结文档**：
   - 游戏开发历程的完整回顾
   - 已实现功能的详细列表
   - 技术架构的清晰描述
   - 当前游戏的优势和限制分析

2. **伏笔细节文档**：
   - 分类清晰的伏笔列表
   - 每个伏笔的详细描述
   - 伏笔的后续发展方向
   - 伏笔的揭示计划

3. **后续开发建议**：
   - 基于伏笔的剧情扩展建议
   - 技术实现的优化方向
   - 游戏机制的改进建议
   - 测试和部署的注意事项

## 优先级

- **高优先级**：上下文总结、核心伏笔提取
- **中优先级**：伏笔设计与规划
- **低优先级**：详细的文档整理

## 时间规划

- 上下文总结：1-2小时
- 伏笔细节提取：2-3小时
- 伏笔设计与规划：3-4小时
- 文档整理与输出：1-2小时

## 输出格式

- 使用Markdown格式编写文档
- 结构化的章节和子章节
- 清晰的列表和表格
- 适当的注释和说明

## 注意事项

- 确保所有伏笔都与游戏核心主题一致
- 避免过度设计，保持伏笔的自然性
- 考虑伏笔的可实现性和技术限制
- 保持文档的可读性和可维护性
